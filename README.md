# 프로젝트개요 / 25-12-21 기준 (지속 업데이트 예정)

## Entity
**BinaryContent**
- 따로 외래키 지정 없음. 파일만 담당함.
- 프로젝트 전체의 첨부파일을 담당하는 엔티티. 
- 유저의 프로필 사진, 메세지의 첨부파일 등에서 사용중. 
- 첨부파일이 있는 엔티티에서 BinaryContent의 ID를 참조해 매핑 중. 1:N

**Channel**
- 따로 외래키 지정 없음. 채널만 담당함.
- 채팅방만 담당함.
- Public(공개), Private(비공개) 두 개의 타입으로 관리 중
- 현재 비공개 채널 개설시 참여자 정보만 받는 중. 비공개 채널의 이름, 설명 없음. 참여자 이름으로 만드는 중

**Message**
- 유저의 ID, 채널의 ID를 외래키로 지정 중.
- 프로젝트 전체의 메세지 저장함. 유저/채널별 엔티티 따로 없음
- 작성자, 메세지 올라간 채널, 첨부파일 목록 관리 중.

**User**
- 유저 담당하는 엔티티
- BinaryContent(전체 파일)에서 프로필 관련 파이 ID 참조중

**ReadStatus**
- 각 체널별, 유저가 마지막으로 읽은 메세지 시간을 관리 중.
- 각 채널의ID, 유저의ID를 외래키로 만듦
- 이걸로 각 채널에 입장해 있는 유저목록도 같이 조회 가능. 
  - 읽은 시간이 있다면 채팅방에 참석해 있다는 것으로 간주
  - 각 채팅방 별로 메세지 안읽은 시간 표시 예정. 방금전, 5분전 등등
  - 추후 전체 메세시 조회, 채널ID비교, 시간 비교해서 각 채널 안읽은 메세지 갯수 파악 가능

**UserStatus**
- 유저가 온라인 중인지 체크. 5분간격으로
